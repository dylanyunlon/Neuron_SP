### 5. 上一轮管理者的错误分析

> 为什么会出现"一个 Claude 一个 hetero_xxx.py"？

根本原因是 **任务粒度设计错误**。管理者把 Megatron 的 commit 列表当成了任务列表，让每个 Claude 按 commit 编号生成对应的文件。这种 1:1 映射忽略了一个事实：一个模块的正确实现需要理解该模块从 first commit 到 HEAD 的完整演化，而不是单个 commit 的差异。加上没有 API contract（没有预定义接口），每个 Claude 自行发明接口，导致互不兼容。

自动化脚本证据：
- 6月13日 02:27-02:28 之间 15 个 commit（每个 1-2 秒）
- 同一个 Megatron commit 被多个 Claude 重复迁移 4 次
- 无去重、无 import 检查

当前的纠正措施（最近 40 个 commit）已经走上正轨：
- 先出 ARCHITECTURE.md 定义接口
- 按模块而非按 commit 分工
- 每个 commit 修改已有文件而非创建新文件
- 用 desloc_engine.py 的 import 作为"活"验证
