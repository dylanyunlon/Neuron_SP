# Neuron_SP Architecture Audit — 2026-06-30

## Executive summary

The Megatron integration is **on track** after today's commits.  
`deepspeed/core/` contains ~20,700 lines of real, imported, tested code across 9 subsystems.  
The "mirrors Megatron" disaster (161 dead files, 150K dead lines) has been mostly cleaned.  
7 dangling imports from the cleanup commit were just fixed (would crash on `import deepspeed`).

## Commit history forensics

| Wave | Date | Count | What happened |
|------|------|-------|---------------|
| SKIP loop | Jun 13 02:27–03:xx | 2,612 | Automated for-loop SKIPped trivial Megatron commits |
| mirrors loop | Jun 18 01:38–04:23 | 161 | Each non-SKIP → `hetero_xxx.py`, commit msg templated, code never imported |
| cleanup | Jun 30 | 1 | `0b3444db` removed 7 dead files (5,180 lines) |
| dangling fix | Jun 30 | 1 | `5895a1a0` guarded 7 bare imports of deleted files |

**Root cause of the 150K dead code**: The manager Claude ran a for-loop that mapped each Megatron commit to a separate `hetero_xxx.py` file, without checking if the file would be imported. Multiple Claudes got overlapping commit ranges → duplicate files (e.g. M3123 appeared 4 times). No architecture design was done first, so each Claude invented its own interfaces.

**Correct approach** (now in effect): Read the Megatron module's full evolution (not individual commits), then modify existing `deepspeed/core/` files to match Megatron's patterns while preserving DES-LOC extensions.
